## User Story 4.1
1. 管理員登入後進入管理頁，新增社團（代碼/名稱/分類必填）。
2. 任一必填留空，按下儲存應阻擋並顯示紅色錯誤。
3. 對既有社團執行「停用/隱藏」，前台社團列表不再顯示，但後台仍可查到。
4. 發布置頂公告後，首頁最上方應出現公告卡，且樣式與一般活動明顯不同。
5. 驗證同社團「社長/公關」主要幹部帳號最多兩個，超過時應被阻擋。

## User Story 2.1
1. 幹部登入後只能管理自己的社團。
2. 更新社團時，介紹內文/社課時間/聯絡信箱/地點為必填。
3. 留空必填欄位時，欄位下方出現紅字「此為必填欄位」。
4. 儲存成功後，打開社團詳情頁，資訊立即更新。
5. 幹部編輯時可看到「更新預覽」區塊。

## User Story 2.2
1. 建立活動時，活動名稱/日期時間/地點/內容為必填。
2. 任一必填欄位空白，不能發布。
3. 發布成功後，社團詳情「近期活動」可看到新活動。
4. 過期活動不應出現在前台清單，未過期活動依活動日期由近到遠。

## User Story 1.1
1. 社團列表頁可選分類並點熱門標籤。
2. 同時選擇分類與多個標籤時，結果為 OR 聯集。
3. 若無結果，顯示「目前無符合此條件的社團，請嘗試其他關鍵字」。

## User Story 1.5
1. 幹部更新社團資料後，社團詳情頁顯示「最後更新時間：YYYY-MM-DD」。
2. 幹部發布新活動後，社團最後更新時間應同步更新。

## User Story 1.3
1. 學生在社團頁點追蹤，按鈕變為「已追蹤」並改變樣式。
2. 再次點擊可取消追蹤，樣式回復。
3. 個人頁動態牆顯示追蹤社團未過期活動，依發布時間新到舊。
4. 若未追蹤任何社團，顯示空狀態文案與「前往探索社團」按鈕。
