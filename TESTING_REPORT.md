## 📝 **詳細問題描述**

### BUG-001: admin.php 第53行重複代碼錯誤
**發現日期**: 2026年3月21日
**發現者**: 系統語法檢查
**問題描述**:
```
在 admin.php 文件中，updateClubStatus 方法被重複定義了兩次，
第一個方法缺少結束的大括號和成功響應消息。
```
**重現步驟**:
1. 打開 `backend/api/admin.php`
2. 查看第53行附近代碼
3. 發現重複的方法定義

**預期結果**: 方法應該只有一個正確的定義
**實際結果**: 方法被重複定義，導致語法錯誤
**修復方案**: 刪除重複代碼，確保方法正確結束
**修復狀態**: ✅ 已修復
**修復日期**: 2026年3月21日
**驗證者**: 開發團隊

### BUG-002: upload API 授權/DB 介面不一致
**發現日期**: 2026年4月2日
**發現者**: 靜態程式碼審查
**問題描述**:
```
upload.php 原本以 new Auth() 與物件屬性方式存取授權與資料庫，但實際 Auth 類別為靜態工具類，
資料庫也應透過 Database 單例取得。
```
**修復方案**: 改為使用 Auth::isLoggedIn()、Auth::getCurrentUserId() 與 Database::getInstance()
**修復狀態**: ✅ 已修復

### BUG-003: events 查詢參數契約不一致
**發現日期**: 2026年4月2日
**發現者**: 靜態程式碼審查
**問題描述**:
```
前端活動列表主要送出 club 與 search，但後端原本只認 club_id，容易造成篩選條件失效。
```
**修復方案**: 後端兼容 club 與 club_id，前端補上 search 參數
**修復狀態**: ✅ 已修復
